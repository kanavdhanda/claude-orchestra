## Lessons Learned

- When a user says "fix X" and the chosen fix is deleting a behavior, that's
  an interpretation, not a given — confirm what's actually broken (the
  *visibility* of routing) before removing the mechanism itself (the
  routing). Got this wrong once this session and had to revert.
- `python3 /path/to/pkg/module.py` and `python3 -m pkg.module` differ in
  `sys.path[0]`: the former only adds the script's own directory, not the
  package's parent. Any LaunchAgent/cron invocation of an in-package script
  needs `-m`, not a direct file path, or it'll silently `ModuleNotFoundError`
  on any `from pkg import ...` inside it.

## Context budget: why sessions start near 113k tokens (57%) and grow to ~146k after compaction

Measured directly this session (`tiktoken`, `cl100k_base`):
- Global + project `CLAUDE.md` + `MEMORY.md`: ~800 tokens combined — negligible.
- `~/.claude.json` (session/project state file): ~22,200 tokens.
- The base system prompt (identity, tool-use rules, safety rules): a few
  thousand tokens.
- The dominant cost is MCP tool schemas: this session alone has ~15 MCP
  servers connected (arxiv, claude-in-chrome, chrome-devtools, context7,
  github, google-drive, ide, serena, ast-grep, design-sync, remote-trigger,
  cron, etc.), together defining on the order of 90 tool functions, several
  with large verbose JSON schemas (Bash, Monitor, DesignSync, the GitHub
  tool family, browser tools). That schema payload is resent as context on
  every turn and is what actually explains ~113k of the ~200k window —
  CLAUDE.md/memory are rounding errors by comparison.
- Post-compaction the number goes *up*, not down, because compaction only
  replaces old raw turns with a shorter summary — it does not shrink the
  fixed tool-schema baseline (~90-95k) which is always present. On top of
  that baseline: the summary text itself costs tokens (a few thousand for a
  detailed one), and every tool call/result made since resuming (file reads,
  bash output, edits) keeps accumulating until the next compaction. 113k
  (fresh) → 146k (mid-session, post-compaction) is consistent with baseline
  + summary + this session's own resumed work, not a leak or bug.
- Actionable lever: the tool-schema tax scales with *how many MCP servers
  are connected*, not with project size. Disconnecting unused MCP servers
  for a given session is the only way to meaningfully lower the 113k floor.
